# W3D4：配置文件与路由 — 总结

## 一、课程内容

- 配置驱动: 参数从代码移到 JSON 配置文件
- JSON 格式: 对象、数组、字符串、数字、布尔
- 配置校验: host/port/document_root/log level/routes 启动前验证
- 路由注册表: handler 名称 → 函数指针映射
- 路由分发: method + path 精确匹配 → handler → 静态文件 → 404/405

## 二、Webserver V1.4：配置驱动

### 设计思路

将 V1.3 中写死在代码里的监听地址、端口、网站根目录、日志和路由
全部移入 JSON 配置文件 config/server.json.
嵌入 cJSON 解析库, 启动时加载 JSON 并逐字段校验,
校验通过后才 bind/listen/进入事件循环.
路由使用 method + path 作为键, handler 名称通过注册表查找函数.

### 新增文件

| 文件 | 说明 |
|------|------|
| `include/cJSON.h` | cJSON 头文件 |
| `src/cJSON.c` | cJSON 解析器实现 |
| `config/server.json` | JSON 配置文件 |

### 修改文件

| 文件 | 改动 |
|------|------|
| `include/config.h` / `src/config.c` | 重构: key=value → JSON 解析, 新增 routes 数组和校验 |
| `src/main.c` | 从 config/server.json 加载, 传递 routes 到服务器 |
| `include/epoll_server.h` / `src/epoll_server.c` | 新增 routes 参数 |
| `include/request_handler.h` / `src/request_handler.c` | 新增配置路由分发 (method+path 匹配 → handler) |
| `Makefile` | +cJSON.o |

### 修复的 BUG

| BUG | 修复 |
|-----|------|
| POST 路由被同路径 GET 路由 405 检查拦截 | 405 判断移到循环外 |
| cJSON.h 预处理器语法错误 | extern "C" 分行 |

### 测试

```bash
./build/mini_web_server config/server.json serve-http
GET /users   → 200 (users_get)
POST /users  → 200 (users_create)
PUT /users   → 405
GET /        → 200 (静态文件)
```
