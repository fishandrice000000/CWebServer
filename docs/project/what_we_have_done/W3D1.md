# W3D1：HTTP 协议概述 — 总结

## 一、课程内容

- HTTP 协议: 请求-响应模式, 请求/响应消息结构
- URL 组成: 协议、主机、端口、路径
- HTTP 请求: 请求行 (方法+URL+版本), 请求头, 空行, 请求体
- HTTP 响应: 状态行, 响应头, 空行, 响应体
- GET vs POST: 参数位置、安全性、数据量
- HTTP 状态码: 2xx 成功, 4xx 客户端错误, 5xx 服务器错误
- 日志系统: 系统日志 (四级), 访问日志 (请求记录)

## 二、训练与项目：完整 HTTP 服务器

### 设计思路

在 W2D5 epoll 事件循环基础上, 用完整的 HTTP 协议解析替换简单的
read_line → handle_http_request 模式.

每个客户端维护独立的接收缓冲区 (8KB), recv 数据后判断请求完整性
(\r\n\r\n 确定头部结束, Content-Length 确定请求体长度), 完整后调用
http_parse_request 解析请求行/头/体, http_handle_request 路由处理,
最后记录访问日志并清理连接.

### 修改文件

| 文件 | 改动 |
|------|------|
| `include/log.h` / `src/log.c` | 日志四级 (DEBUG/INFO/WARNING/ERROR), 访问日志独立文件 |
| `include/request_handler.h` / `src/request_handler.c` | 新增 http_parse_request, http_handle_request, http_get_header |
| `src/epoll_server.c` | 重构为 per-client 缓冲区 + 完整性判断 + 完整 HTTP 处理流程 |
| `src/main.c` | 新增 serve-http 子命令, 访问日志初始化/关闭 |

### 新增路由

| 路由 | 方法 | 状态码 | 说明 |
|------|------|------|------|
| GET / | GET | 200 | 返回 HTML 页面 |
| GET /hello | GET | 200 | Hello, Web! |
| GET /users/<name> | GET | 200 | 用户查找 |
| POST /echo | POST | 200 | 回显请求体 |
| GET /missing | GET | 404 | Not Found |
| 其他未知路径 | GET | 404 | Not Found |
| 非 GET/POST | 任何 | 405 | Method Not Allowed |

### 修复的 BUG

| BUG | 修复 |
|-----|------|
| 请求行长度检查引用 sizeof(req->method) 而非 sizeof(first_line) | 改为 sizeof(first_line) |
| 访问日志状态码推断不准确 (404 显示为 200) | http_handle_request 新增 status_code 输出参数 |
