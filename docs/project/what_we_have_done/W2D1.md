# W2D1：Linux 网络编程基础 — 上午总结

## 一、课程内容

- 网络协议分层: OSI 七层 vs TCP/IP 四层模型
- TCP/IP 通信过程: 逐层封装与解封装, C/S 架构
- Socket API: socket / bind / listen / accept / connect / send / recv
- 地址结构: sockaddr_in (sin_family, sin_port, sin_addr)
- 字节顺序: 网络字节序 (大端) vs 主机字节序, htons/htonl/ntohs/ntohl
- IP 地址转换: inet_addr / inet_ntoa / inet_pton

## 二、训练：TCP 文本聊天程序

**路径:** `training/w2d1/chat/`

| 组件 | 流程 | 说明 |
|------|------|------|
| 服务器 | socket → bind → listen → accept → recv/send | 等待客户端连接, 收到消息后回复, exit 退出 |
| 客户端 | socket → connect → send/recv | 命令行指定 IP:端口, 连接后双向文字通信, exit 退出 |

**构建与运行:**

```bash
make
# 终端 1:
./server 8080
# 终端 2:
./client 127.0.0.1 8080
```

## 三、Webserver V0.6：TCP 网络服务模块（下午）

### 设计思路

从 V0.6 开始, 请求不再来自 `requests/*.req` 文件, 而是来自真实的 TCP 连接.
服务器监听 `config/server.conf` 中配置的 host:port, 接收 HTTP 请求行并返回响应.
用 curl 模拟客户端测试.

### 新增文件

| 文件 | 说明 |
|------|------|
| `include/tcp_server.h` | TCP 服务器接口 |
| `src/tcp_server.c` | socket → bind → listen → accept, SO_REUSEADDR, 调用 handle_http_request |
| `test/test_day06.sh` | 自动化测试 5 项 (curl 测试 /hello, /users, /not-exist, 404) |

### 修改文件

| 文件 | 改动 |
|------|------|
| `include/request_handler.h` | 新增 `handle_http_request(int conn_fd, UserNode*)` |
| `src/request_handler.c` | 新增 `read_line` (逐字节读 socket) + `handle_http_request` (解析 HTTP 请求行 → 路由 → dprintf 响应) |
| `src/main.c` | 新增 `serve` 子命令 |
| `Makefile` | V0.5→V0.6, +`tcp_server.o`, +`test06` |

### 修复的 BUG

| BUG | 修复 |
|-----|------|
| 400/404 响应缺少 Content-Length | 补 `Content-Length: 0` |
| FOUND/NOT_FOUND 缺少 Content-Length | snprintf 计算正文长度再 dprintf |
| /hello Content-Length 硬编码 12 | 改为 strlen 动态计算 |
| 测试端口冲突 | setsockopt(SO_REUSEADDR) + 测试脚本独立启停 |
