## 四、Webserver V0.4：多进程请求处理（下午）

### 设计思路

父进程扫描 `requests/` 下的 `.req` 文件, 每个文件 fork 子进程处理, 结果写入 `outputs/`. 父进程通过 waitpid 等待所有子进程结束. 用户数据在父进程中加载一次, 通过指针传递给子进程, 避免重复 I/O.

### 新增文件

| 文件 | 说明 |
|------|------|
| `include/request_handler.h` | 请求处理接口 (`handle_request`) |
| `include/process_server.h` | 多进程服务接口 (`process_requests`) |
| `src/request_handler.c` | 解析 GET /hello, /users/<name>, /not-found, 写输出 |
| `src/process_server.c` | 扫描 reqs, fork, waitpid, 日志 |
| `test/test_day04.sh` | 自动化测试 6 项 |
| `requests/*.req` | hello / user_find / missing |
| `outputs/.gitkeep` | 输出目录占位 |

### 修改文件

| 文件 | 改动 |
|------|------|
| `src/main.c` | 新增 `process` 子命令 |

### 修复的 BUG

| BUG | 修复 |
|-----|------|
| fork 前 fflush 缺失 | `fflush(NULL)` 刷新所有流 |
| snprintf 缓冲区过小 | PATH_SZ/MSG_SZ → 1024, `%.512s` 限制 |
| 子进程重复加载 CSV | 父进程加载一次, 指针传递 |
