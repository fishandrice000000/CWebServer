## 二、训练与 Webserver V0.8：线程池 TCP 网络服务

今日训练内容即为 V0.8 的线程池实现, 训练和项目合二为一.

### 设计思路

从 V0.7 的多进程 fork 模式升级为固定大小的线程池模式:
主线程 accept → 将 client_fd 放入循环队列 → worker 线程从队列取出并调用
handle_http_request → 处理完毕后 worker 关闭 fd。

使用 mutex + 双条件变量 (not_empty / not_full) 实现线程安全的生产者-消费者模型。
worker 通过 while(count==0) 检查队列状态, 避免信号丢失和虚假唤醒。

### 与 V0.5 thread_server 的对比

| | V0.5 thread_server | V0.8 thread_pool |
|------|------|------|
| 数据来源 | 目录扫描 .req 文件 | TCP accept |
| 队列内容 | 文件路径字符串 | socket fd |
| 处理函数 | handle_request (文件→文件) | handle_http_request (socket 直接回复) |
| 封装程度 | static 全局变量, 耦合在主流程 | 独立模块, 可复用 |

### 修复的 BUG

| BUG | 修复 |
|-----|------|
| 错误恢复路径条件变量销毁不完整 | 三处路径补 not_full 销毁和广播; init 拆分为独立判断 |
