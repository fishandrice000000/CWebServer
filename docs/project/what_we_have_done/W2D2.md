# W2D2：Linux 网络编程基础 — 上午总结

## 一、课程内容

- 多进程并发服务器基本架构: fork 模式
- TCP 回射服务器/客户程序: 正常启动与终止流程
- Posix 信号处理: signal / sigaction
- 处理 SIGCHLD 信号: 僵尸进程产生原因, wait / waitpid 回收
- accept 被信号打断: EINTR 处理
- 程序运行异常处理: 连接夭折、服务器进程终止、SIGPIPE 信号、主机崩溃/重启/关机
- 数据格式: 文本串 vs 二进制结构

## 二、训练：多客户端 TCP 文本聊天程序

**路径:** `training/w2d2/chat/`

| 组件 | 流程 | 说明 |
|------|------|------|
| 服务器 | socket → bind → listen → signal(SIGCHLD) → accept 循环 → fork | 每客户端 fork 子进程处理, 父进程回收子进程, accept EINTR 重试 |
| 客户端 | socket → connect → send/recv | 命令行指定 IP:端口, 连接后双向文字通信, exit 退出 |

**关键改进 (对比 W2D1):**

| 特性 | 实现 |
|------|------|
| 多客户端并发 | fork 子进程, 父进程继续 accept |
| 僵尸进程回收 | signal(SIGCHLD, sig_chld) + waitpid(-1, &stat, WNOHANG) |
| accept 中断处理 | errno == EINTR 时 continue |

**构建与运行:**

```bash
make
# 终端 1:
./server 8080
# 终端 2 / 终端 3 (多个客户端):
./client 127.0.0.1 8080
```

## 三、Webserver V0.7：多进程 TCP 网络服务（下午）

### 设计思路

从 V0.6 的单连接 TCP 服务器升级为多进程并发: 父进程在 accept 循环中为每个客户端 fork 子进程处理 HTTP 请求。注册 SIGCHLD 回收子进程, 忽略 SIGPIPE 防止客户端断开时崩溃, accept 被信号打断时检查 EINTR。

### 新增文件

| 文件 | 说明 |
|------|------|
| `include/tcp_fork_server.h` | 多进程 TCP 服务器接口 |
| `src/tcp_fork_server.c` | socket → bind → listen → accept 循环 → fork, SIGCHLD 信号异步安全处理, EINTR, SIGPIPE, max_clients |
| `test/test_day07.sh` | 自动化测试 7 项 (编译、并发请求、路由正确性、僵尸进程、SIGPIPE) |

### 修改文件

| 文件 | 改动 |
|------|------|
| `src/main.c` | 新增 `serve-fork` 子命令, include tcp_fork_server.h |
| `Makefile` | V0.6→V0.7, +tcp_fork_server.o, +test07 |

### 修复的 BUG

| BUG | 修复 |
|-----|------|
| sig_chld 中调用 printf/log_info (非异步安全) | 改为 volatile sig_atomic_t 标志位, 回收逻辑移到循环末尾和循环后 |
| test_day07 中 not-found 检查假通过 | 合并到并发批次, 用 -w '%{http_code}' 替代 \|\| echo '404' |
| max_clients break 后 SIGCHLD 漏回收 | 循环后补 got_sigchld 检查 + 阻塞式 waitpid 收尾 |
| SIGPIPE 测试 max_clients=1 导致误判 | 改为 max_clients=2, 服务器不会因配额满而退出 |
