# W3D2：HTTP 处理静态资源 — 总结

## 一、课程内容

- URL 路径映射到网站根目录: / → /index.html, 多级目录
- 路径规范化: 去除查询参数, 拒绝 .. 目录穿越
- 安全校验: realpath() 限定 document root, S_ISREG 只发送普通文件
- MIME 类型: 扩展名映射 (html/css/js/jpeg/png/gif/ico/json)
- stat() 获取文件元数据: st.st_mode (类型), st.st_size (长度)
- 分块发送: 固定缓冲区, send_all() 处理部分发送和 EINTR
- 状态码: 200/400/403/404/405/500
- errno 映射: ENOENT→404, EACCES→403

## 二、Webserver V1.2：静态资源文件服务

### 设计思路

在 V1.1 epoll HTTP 服务器上新增静态文件服务层.
GET 请求先检查旧版路由 (/hello, /users/<name>), 未匹配则映射到 www/ 目录.
路径经过去查询参数 → ../拒绝 → realpath 校验 → stat 检查 → MIME 映射 → 分块发送.
保留 POST /echo 和非 GET/POST 的 405 处理.

### 新增文件

| 文件 | 说明 |
|------|------|
| `www/index.html` | 首页 (引用 CSS/JS/图片/favicon) |
| `www/css/style.css` | 样式表 |
| `www/js/app.js` | JavaScript |
| `www/img/logo.jpeg` | 图片资源 |
| `www/favicon.ico` | 网站图标 |

### 修改文件

| 文件 | 改动 |
|------|------|
| `include/request_handler.h` / `src/request_handler.c` | 新增 MIME 映射、send_all、normalize_path、validate_path、serve_static_file; http_handle_request 新增 doc_root 参数 |
| `include/epoll_server.h` / `src/epoll_server.c` | 新增 doc_root 参数传递 |
| `src/main.c` | 传入 config.root 给 epoll_server_run |

### 修复的 BUG

| BUG | 修复 |
|-----|------|
| 静态文件服务导致 /hello 和 /users/ 路由丢失 | 在 GET 分支优先检查旧版路由, 未匹配再回退静态文件 |

### 测试

```bash
curl /               # 200, HTML 首页
curl /hello          # 200, Hello, Web!
curl /css/style.css  # 200, text/css
curl /img/logo.jpeg  # 200, image/jpeg
curl --path-as-is /../etc/passwd  # 403
```
