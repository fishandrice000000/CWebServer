# W3D5：HTTP Basic 认证 — 总结

## 一、课程内容

- 认证 vs 授权: 401 Unauthorized vs 403 Forbidden
- Basic 认证流程: 401 → WWW-Authenticate → Authorization → Base64 解码 → 凭据校验
- Base64: 编码格式、= 填充、解码实现
- 安全边界: Base64 不是加密、HTTPS 必需、日志不输出凭据

## 二、Webserver V1.5：Basic 认证

### 设计思路

在 V1.4 路由分发基础上增加认证层. 配置路由的 auth 字段为 "basic" 时,
在 handler 之前调用 basic_auth_check:
读取 Authorization 请求头 → 验证 Basic scheme → Base64 解码 →
按冒号拆分 username:password → 与预设凭据 (student/lab123) 比较.
通过则执行 handler, 失败则返回 401 + WWW-Authenticate.

### 修改文件

| 文件 | 改动 |
|------|------|
| `include/config.h` | route_config_t 新增 auth 字段 |
| `src/config.c` | JSON 解析新增 auth 字段 (可选) |
| `src/request_handler.c` | 新增 base64_decode、basic_auth_check、secured_get handler |
| `config/server.json` | 新增 /secured 受保护路由 (auth: basic) |

### 修复的 BUG

| BUG | 修复 |
|-----|------|
| Base64 解码在 = 处截断, student:lab123 → student:lab1 | 循环后处理残留不完整组 |
| secured_get handler 缺失 | 新增 |

### 测试

```bash
curl /                 # 200 (公开)
curl /secured          # 401 (无凭据)
curl -u student:wrong /secured   # 401
curl -u student:lab123 /secured  # 200
```
