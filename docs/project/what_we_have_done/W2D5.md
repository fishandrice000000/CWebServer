# W2D5：epoll 并发模型 — 总结

## 一、课程内容

- epoll 原理: 红黑树 + 就绪队列, LT vs ET 模式
- epoll API: epoll_create / epoll_ctl (ADD/MOD/DEL) / epoll_wait
- struct epoll_event: events (EPOLLIN/EPOLLOUT/EPOLLET) + data.fd
- select vs poll vs epoll 对比: 注册方式、遍历开销、数量限制
- I/O 模式: 阻塞/非阻塞/I/O 多路复用/信号驱动/异步

## 二、训练：EpollServer + TCP Client

**路径:** `training/w2d5/epoll/`

| 组件 | 流程 | 说明 |
|------|------|------|
| EpollServer | socket→bind→listen→epoll_create→epoll_ctl(ADD listen_fd)→epoll_wait 循环→accept/recv | epoll LT 模式, \n 分帧, quit 退出, epoll_ctl DEL+close |
| TCP Client | socket→connect→send | quit 退出 |

**与 W2D4 SelectServer 对比:**

| | SelectServer | EpollServer |
|------|------|------|
| 创建实例 | 不需要 | epoll_create(1) |
| 注册 fd | FD_SET 每次重拷 | epoll_ctl ADD 只一次 |
| 等待事件 | select 修改传入集合 | epoll_wait 输出数组 |
| 获取就绪 | 遍历全部 FD_ISSET | 直接遍历 events[] |
| 删除 fd | FD_CLR | epoll_ctl DEL |
| 数量限制 | 1024 | 无限制 |

## 三、Webserver V1.0：epoll 高并发 Web Server（下午）

### 设计思路

使用 epoll LT 模式替代 select/poll/线程池, 实现单线程高并发 HTTP 服务器。
epoll_create 后注册 listen_fd, epoll_wait 循环中区分新连接和 HTTP 请求。
新连接 accept 后通过 epoll_ctl ADD 加入监控, HTTP 请求交给已有的 handle_http_request 处理,
完毕后 epoll_ctl DEL + close。
