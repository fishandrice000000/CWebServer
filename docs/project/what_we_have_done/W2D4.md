# W2D4：面向网络的并发编程 — select 模型

## 一、课程内容

- I/O 模式: 阻塞 I/O、非阻塞 I/O、I/O 多路复用
- select 函数模型: 工作原理、fd_set 位图、FD_ZERO/FD_SET/FD_ISSET/FD_CLR
- select 的缺点: FD_SETSIZE=1024 限制、内核/用户空间内存拷贝、O(n) 遍历
- poll 函数模型: pollfd 结构, 无固定长度限制但仍需遍历
- epoll 概述: 红黑树 + 就绪队列, 解决 select/poll 的性能瓶颈

## 二、训练：SelectServer + TCP Client

**路径:** `training/w2d4/select/`

| 组件 | 流程 | 说明 |
|------|------|------|
| SelectServer | socket → bind → listen → FD_ZERO → FD_SET(listen_fd) → select 循环 → FD_ISSET 检查 → accept/recv → FD_CLR | 单线程多路复用, \n 分帧, quit 清理 |
| TCP Client | socket → connect → send/recv | 命令行指定 IP:端口, quit 退出 |

**关键特性:**

| 特性 | 实现 |
|------|------|
| I/O 多路复用 | select 同时监控 listen_fd + 所有 client_fd |
| 消息分帧 | 每客户端独立缓冲区, memchr 查找 \n, memmove 整理残留数据 |
| 客户端退出 | quit\n 或 recv()==0 时 FD_CLR + close + 更新计数 |
| 单线程 | 不使用多线程/多进程/线程池 |

**构建与运行:**

```bash
make
# 终端 1:
./SelectServer 127.0.0.1 8888
# 终端 2/3/4:
./client 127.0.0.1 8888
```
