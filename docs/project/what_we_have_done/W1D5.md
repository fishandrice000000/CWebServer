# W1D5：Linux 多线程并发编程 — 上午总结

## 一、课程内容

- 线程创建: `pthread_create` 的参数与用法, 线程启动例程设计
- 线程终止: `pthread_exit` vs 从启动例程 return
- 线程等待: `pthread_join` 阻塞等待子线程结束
- 线程数据共享: 全局变量, 堆内存跨线程可见, 但需要同步保护
- 互斥量: `pthread_mutex_init/lock/unlock/destroy`, `trylock` 非阻塞加锁
- POSIX 信号量: `sem_open/wait/post/unlink`
- 死锁: 哲学家同时拿起一边筷子形成循环等待


## 三、Webserver V0.5：多线程请求处理（下午）

### 设计思路

在 V0.4 多进程基础上改为多线程模型. 主线程扫描 `requests/`, 将请求路径加入共享环形队列; 创建 N 个 worker 线程, 从队列取任务用 `handle_request` 处理. 互斥量保护队列访问, 条件变量实现高效等待 (无忙轮询). 用户链表主线程加载一次, 多线程只读共享 (无需加锁).

### 新增文件

| 文件 | 说明 |
|------|------|
| `include/thread_server.h` | 接口声明 |
| `src/thread_server.c` | 共享队列 + mutex + cond, worker 线程, 主线程分发 |
| `test/test_day05.sh` | 自动化测试 8 项 |

### 修改文件

| 文件 | 改动 |
|------|------|
| `src/main.c` | 新增 `threaded <n>` 子命令 |
| `Makefile` | V0.4→V0.5, +`LDFLAGS=-lpthread`, +`thread_server.o`, +`test05` |

### 与 V0.4 多进程对比

| 特性 | V0.4 (进程) | V0.5 (线程) |
|------|-----------|-----------|
| 内存 | 独立地址空间, fork COW | 共享地址空间 |
| 通信 | 文件/管道 | 共享队列 (mutex+cond) |
| 任务分配 | 父进程静态分配 | 线程动态竞争取任务 |
| 数据共享 | 需 IPC 或 fork 前加载 | 直接共享指针, 只读无锁 |
| 开销 | 进程创建较重 | 线程创建较轻 |

### 修复的 BUG

| BUG | 修复 |
|-----|------|
| opendir 失败泄漏 mutex/cond | init 移到 opendir 成功后 |
| malloc 未检查 NULL | 两处加 NULL 检查 + 错误路径清理 |
