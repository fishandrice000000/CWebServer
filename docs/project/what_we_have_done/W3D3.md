# W3D3：HTTP 处理 GET 和 POST 请求 — 总结

## 一、课程内容

- GET 和 POST 的核心差异: 参数位置 (URL vs 请求体), HTTP 语义
- 查询参数解析: ? 分隔路径和查询, & 分隔参数, = 分隔名值
- URL 解码: %HH → 字符, + → 空格
- POST 表单: application/x-www-form-urlencoded, Content-Length
- HTML 转义: & < > " ' 防注入
- 参数校验: class 4位数字, keyword 1~64字节
- 状态码: 200/400/403/404/405/413/415/500

## 二、Webserver V1.3：GET/POST 动态查询

### 设计思路

在 V1.2 静态文件服务基础上新增 /search 动态查询路由.
GET /search 返回 HTML 查询表单; GET /search?class=...&keyword=... 解析 URL 参数;
POST /search 解析 application/x-www-form-urlencoded 请求体.
两者共用同一个数据文件查询逻辑, 搜索 data/<class>.txt 并生成 HTML 结果页.

### 新增文件

| 文件 | 说明 |
|------|------|
| `data/2011.txt` | 示例数据 (学号\t姓名\t性别) |

### 修改文件

| 文件 | 改动 |
|------|------|
| `src/request_handler.c` | 新增 url_decode, get_query_param, html_escape, build_search_form, build_search_result, handle_search |

### 修复的 BUG

| BUG | 修复 |
|-----|------|
| get_query_param static 缓冲区两次调用互相覆盖 | 每次调用后立即 strncpy |
| /search 路由被静态文件服务拦截 | 移到 GET 分支顶部 |
| handle_search 错误路径无 HTTP 响应 | goto send_error |

### 测试

```bash
curl /search                        # 200, 查询表单
curl '/search?class=2011&keyword=孙七'  # 200, 搜索结果
curl -X POST -d 'class=2011&keyword=李四' /search  # 200, 搜索结果
```
