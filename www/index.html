功能特性

⚙

JSON 配置驱动

服务器端口、日志级别、文档根目录、路由表全部由 server.json 配置，启动时自动校验

☰

路由分发

支持方法 + 路径精确匹配，可注册多个路由，自动返回 405 Method Not Allowed 或 404 Not Found

🔒

Basic 认证

路由级 HTTP Basic Authentication，Base64 凭据解码，未授权返回 401 + WWW-Authenticate

📄

静态文件服务

支持 HTML / CSS / JS / 图片 / 图标等静态资源，自动 MIME 类型识别，路径穿越防护

🔍

动态查询

GET / POST 表单处理，URL 解码 (%HH, +)，查询参数解析，HTML 实体转义防 XSS

📝

访问日志

标准 Apache 格式日志：时间、IP、方法、URL、状态码、字节数、User-Agent，可配置日志级别

技术架构

Reactor 模式 + 非阻塞 I/O + 线程池，单进程处理万级并发连接

        4 配置文件 / 路由生效 → 200
      

      
config/server.json 中定义的路由表驱动请求分发。三个按钮分别验证三种路由来源：

      
    
        
  • 内置路由 — /hello 硬编码在 C 代码中，不经过配置文件，返回纯文本 "Hello, Web!"
    • 
        
  • JSON 配置路由 — GET /users 在 server.json 路由表中定义 {"method": "GET", "path": "/users", "handler": "users_get"}，请求被路由表匹配并交给对应 handler 处理
    • 
        
  • POST 回显 — /echo 也是内置路由，将请求体原样返回，证明 POST 方法解析与消息体读取正常
    •

        6 Basic 认证测试 → 401 / 200
      

      
测试端点 /api/auth-check（与 /secured 认证逻辑相同，返回 JSON，永不弹浏览器登录框）。正确凭据：student / lab123。

      
    
        
  • 用户名留空点击登录 → 401（无凭据）
    • 
        
  • 填入错误密码点击登录 → 401（凭据无效）
    • 
        
  • 填入正确凭据 student / lab123 点击登录 → 200（认证通过）

        7 访问日志 → 不泄露凭据
      

      
每次请求记录：时间、IP、方法、URL、状态码、字节数、User-Agent。认证凭据（Authorization 头、Base64 字符串、密码）不会出现在日志中。点击刷新按钮从服务器实时拉取日志内容。
